14. Основные свойства протокола HTTP:
response, request и из структура
HTTP Request (Запрос HTTP):

HTTP-запрос отправляется клиентом (например, веб-браузером) серверу для запроса определенного ресурса или выполнения определенной операции на сервере. Запрос состоит из нескольких частей:
    Метод (Method): Определяет, какое действие должно быть выполнено с ресурсом. Например, метод GET используется для получения данных, метод POST - для отправки данных на сервер, и так далее.
    URI (Uniform Resource Identifier): URI определяет путь к ресурсу на сервере. Он может включать в себя путь к файлу, параметры запроса и фрагменты.
    Заголовки (Headers): Заголовки представляют собой метаданные запроса, которые могут содержать информацию о клиенте, типе контента, желаемой кодировке и многие другие параметры.
    Тело (Body): Тело запроса содержит данные, которые отправляются на сервер. Это чаще всего используется с методами POST и PUT для передачи данных формы, JSON или других типов данных.
HTTP Response (Ответ HTTP):
HTTP-ответ отправляется сервером клиенту в ответ на запрос. Ответ также состоит из нескольких частей:
    Статус-код (Status Code): Статус-код указывает на результат выполнения запроса и определяет, был ли запрос успешным или произошла ошибка. Например, код 200 означает успешный запрос, а код 404 указывает на то, что ресурс не найден.
    Заголовки (Headers): Как и в запросе, заголовки содержат метаданные ответа, такие как тип контента, кодировка, дата и другие.
    Тело (Body): Тело ответа содержит фактические данные, которые возвращаются клиенту. Например, это может быть HTML-страница, JSON-данные или другие типы содержимого.

Безсостояние (stateless): каждый запрос от клиента к серверу рассматривается как новый.
Механизм для сохранения состояния : cookies

!!! Все методы запросов GET POST PUT DELETE HEAD CONNECTION FRAISE(?) PATCH OPTIONS
GET
    Метод GET запрашивает представление ресурса. Запросы с использованием этого метода могут только извлекать данные.
HEAD
    HEAD запрашивает ресурс так же, как и метод GET, но без тела ответа.
POST
    POST используется для отправки сущностей к определённому ресурсу. Часто вызывает изменение состояния или какие-то побочные эффекты на сервере.
PUT
    PUT заменяет все текущие представления ресурса данными запроса.
DELETE
    DELETE удаляет указанный ресурс.
CONNECT
    CONNECT устанавливает "туннель" к серверу, определённому по ресурсу.
OPTIONS
    OPTIONS используется для описания параметров соединения с ресурсом.
TRACE
    TRACE выполняет вызов возвращаемого тестового сообщения с ресурса.
PATCH
    PATCH используется для частичного изменения ресурса.

17. Группы заголовков HTTP:

Общие заголовки (General headers) — например, Date, Cache-Control.
Заголовки запроса (Request headers) — например, Accept, User-Agent.
Заголовки ответа (Response headers) — например, Server, WWW-Authenticate.
Заголовки сущности (Entity headers) — например, Content-Type, Content-Length. Всё что связано с content это entity

Какие методы идепотентны
Идемпотентность в контексте HTTP означает, что выполнение одного и того же HTTP-запроса несколько раз не должно изменять состояние сервера. То есть даже если вы отправите один и тот же запрос многократно, результат должен быть идентичным, и это не должно вызывать нежелательных или неожиданных изменений на сервере.
Следующие HTTP-методы являются идемпотентными:GET, HEAD, PUT, DELETE, OPTIONS

 Сервер обрабатывает эти запросы, может обращаться к базе данных и возвращает результаты обратно на frontend.

19. Асинхронный запрос — это запрос, отправляемый из браузера на сервер без перезагрузки всей страницы.
 Это позволяет обновлять части страницы без необходимости загружать её целиком.

20. AJAX (Asynchronous JavaScript and XML) — это технология, позволяющая веб-приложениям делать асинхронные запросы к серверу,
 обмениваясь данными в формате XML, JSON или других форматах, и динамически обновлять контент на странице без полной её перезагрузки.

21. HTTP-заголовок Content-Type указывает на медиатип содержимого тела сообщения. Он используется, чтобы определить,
 в каком формате представлены данные (например, text/html для HTML-документов или application/json для данных в формате JSON).

22. Функция require в Node.js используется для импорта модулей или библиотек. Она позволяет загружать и использовать внешние модули или файлы в текущем файле или модуле.

23. Модуль Node.js — это отдельный файл или пакет файлов, содержащий функции или объекты, 
которые можно импортировать и использовать в других файлах с помощью функции require.

24. Node.js built-in modules (или Node.js Core modules) — это модули, включенные в стандартную поставку Node.js,
 и они доступны для использования без дополнительной установки. Примеры таких модулей: fs (работа с файловой системой),
 http (работа с протоколом HTTP), url (анализ URL) и многие другие.

25. В Node.js модуль HTTP предоставляет функциональность для работы с протоколом HTTP, позволяя создавать HTTP-серверы или делать HTTP-запросы.
 Модуль fs обеспечивает работу с файловой системой, позволяя читать, писать, удалять файлы и т. д.
